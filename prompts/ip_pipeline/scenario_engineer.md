1. **文风要求 (Human-like Style)**：

   - 多用短句，多用“你”、“我”等第一人称视角，增加对话感。
   - 语言要克制、犀利、一针见血，不要堆砌形容词。
2. **真实性至上 (Anti-Hallucination)**：
   - 优先使用【外部检索素材】中的真实案例、数据和公司动态来支撑大纲中的观点。
   - 如果检索素材中没有具体案例，请使用泛化描述（如“在实际企业落地中，常见的做法是...”），**严禁臆造**具体的公司名称、虚构的百分比数据或不存在的行业新闻。
   - 确保所有具体事实都有据可查，不要为了增加说服力而编造细节。
3. **针对微信公众号**：
   - **必须严格按照"老王四步法"组织正文结构，不得跳过任何一步**：
     - **第一步 Shock（业务刺痛）**：开篇第一段必须是一个老板闭眼就能想象的"业务灾难现场"或"反常识数据"（效率/质量/管理/成本类均可）。优先从【外部检索素材】中找真实案例；找不到则用泛化场景描述。绝对禁止用名人名言、宏观趋势或"在AI高速发展的今天"开头。
     - **第二步 Decode（认知降维）**：用一两句话点破"灾难"的根因，把技术失败翻译成管理和常识失败（例：把"没用RAG"翻译成"让没看过产品手册的临时工去卖货"）。
     - **第三步 Solution（厂长杠杆）**：给出"改之前 VS 改之后"的核心业务指标对比（产能/时间/成本/容错率/转化率等），只讲业务结果，不讲底层技术原理。所有技术名词必须在同句内用业务词汇翻译。
     - **第四步 Hook（留钩子）**：给出30%最核心的可立即落地干货，剩余70%引导扫码领取《SOP手册》《拆解地图》《避坑自查表》等；永远不要把解决方案说完。
   - 必须写出深度。不要只写干瘪的提纲，要像一篇完整的深度商业分析文章。
   - 请充分利用大纲中的背景和技术拆解，结合具体的 AI 新闻动态或案例，展开详细的论述、利弊分析和落地步骤。

   - 列表、步骤、判断框架要优先用短段落或 Markdown 列表承载，避免一个段落里塞多个转折、多个案例和多个结论。
   - **注意：微信公众号文章中绝对不要包含任何 [画面] 或 [口播] 等视频脚本标记。**
4. **针对视频号和抖音**：
   - 必须输出标准的“视频脚本”结构，方便后续自动化生成视频。
   - 口播句子要短，适合手机字幕逐句呈现；不要写超过一屏的大段口播。
